# ☕ Кофейня — мини-игра на A-Frame + AR

Студентка собирает стаканчики кофе (+5 очков), не задевая однокурсников (−5 очков). Таймер 60 секунд. В конце — результат и уровень Barista. В AR-режиме мини-кофейня появляется на маркере (лист A4).

---

## Требования

- Современный браузер с WebGL (Chrome, Firefox, Edge, Safari).
- Для **AR** — телефон или ноутбук с камерой и **HTTPS** (или localhost).
- Для маркера — распечатанный маркер Hiro или свой pattern (см. ниже).

---

## Как запустить игру (обычный режим)

### 1. Локальный сервер (обязательно для загрузки ресурсов)

Файлы нельзя открывать как `file://` — нужен HTTP-сервер.

**Вариант A — Python 3:**

```bash
cd "путь/к/папке/coffee-game"
python3 -m http.server 8080
```

Открой в браузере: **http://localhost:8080**

**Вариант B — Node.js (npx):**

```bash
cd "путь/к/папке/coffee-game"
npx serve -l 8080
```

Открой: **http://localhost:8080**

**Вариант C — VS Code / Cursor**

Установи расширение "Live Server" и открой `index.html` через "Open with Live Server".

### 2. Управление

- **Мышь** — осмотр сцены, клик по стаканчикам и однокурсникам.
- **W A S D** или **стрелки** — движение камеры.
- Стаканчики: **клик** → +5 очков, объект исчезает.
- Однокурсники: **клик** → −5 очков (очки не уходят ниже 0).
- Через **60 секунд** игра заканчивается, показывается экран с итогом и уровнем Barista.

### 3. Уровни Barista

| Очки | Уровень            |
|------|--------------------|
| 80+  | 🌟 Barista Legend! |
| 50+  | ☕ Barista Pro!    |
| 25+  | 🥤 Barista Junior |
| 0+   | 📚 Стажёр         |

## Как запустить AR-режим (маркер на столе)

AR работает только по **HTTPS** (или `http://localhost`). На телефоне удобнее открыть ту же страницу по сети (например через ngrok или локальный IP, если сервер отдаёт по HTTPS).

### 1. Запуск страницы AR

Тот же локальный сервер, что и выше. Открой в браузере:

**http://localhost:8080/ar.html**

(Если папка сервера — корень проекта, то путь может быть:  
`http://localhost:8080/coffee-game/ar.html` — смотри, откуда сервер раздаёт файлы.)

### 2. Маркер Hiro (проще всего)

1. Скачай и распечатай маркер **Hiro** (чёрно-белый квадрат с узором):
   - [AR.js Hiro marker (PNG)](https://raw.githubusercontent.com/AR-js-org/AR.js/master/data/images/hiro.png)
   - Или открой: [AR.js Marker Training](https://ar-js-org.github.io/AR.js/three.js/examples/marker-training/examples/generator.html) и сохрани изображение маркера Hiro.
2. Наклей на плотный лист или картон (можно на лист A4).
3. Запусти `ar.html`, разреши доступ к камере.
4. Наведи камеру на маркер — на нём должна появиться мини-кофейня со стаканчиками и фигурками.
5. Кликай по стаканчикам (+5) и не кликай по однокурсникам (−5). Очки выводятся в консоль и в подсказке внизу экрана.

### 3. Свой маркер (pattern) для листа A4

Если хочешь использовать **свой** рисунок как маркер (например, логотип на листе A4):

1. Зайди на [AR.js Pattern Marker Generator](https://ar-js-org.github.io/AR.js/three.js/examples/marker-training/examples/generator.html).
2. Загрузи **чёткое чёрно-белое** изображение (высокий контраст, без мелких деталей).
3. Скачай сгенерированные файлы:
   - изображение маркера для печати;
   - файл **.patt** (pattern).
4. Положи файл `.patt` в папку `coffee-game`, например `assets/pattern-marker.patt`.
5. В `ar.html` замени маркер с preset на pattern:

```html
<a-marker id="marker" type="pattern" url="assets/pattern-marker.patt" size="0.2" emitevents="true" ...>
```

Размер `size="0.2"` — ориентировочно сторона маркера в метрах; подгони под реальный размер распечатанного маркера на A4.

### 4. Открыть AR на телефоне

- Убедись, что телефон и компьютер в одной сети.
- Узнай IP компьютера (например, `ipconfig` / `ifconfig`).
- Запусти сервер так, чтобы он был доступен по сети (например, `python3 -m http.server 8080 --bind 0.0.0.0`).
- На телефоне открой в браузере: `http://IP:8080/ar.html` (например, `http://192.168.1.10:8080/ar.html`).

**Важно:** в части браузеров (особенно на iOS) для доступа к камере в AR может требоваться **HTTPS**. Тогда используй туннель (ngrok, localtunnel) с HTTPS или хостинг с SSL.

## AR по GPS-координатам (без маркера)

В проекте есть отдельная страница **AR по геолокации**: объекты (стаканчики и студенты) появляются в реальном мире по координатам GPS вокруг тебя.

### Запуск

1. Открой **http://localhost:8080/ar-gps.html** (тот же сервер, что и выше).
2. Разреши доступ к **камере** и **геолокации** в браузере.
3. Подожди, пока определится GPS (на телефоне лучше выйти на улицу или у окна).
4. После получения координат начнётся та же игра: 60 секунд, стаканчики (+5/+10), студенты (−5). Объекты расставляются в радиусе нескольких метров от твоей текущей позиции.

### Как это устроено

- Используется **AR.js** в режиме **location-based**: компоненты `gps-new-camera` (камера привязана к GPS и компасу) и `gps-new-entity-place` (объекты задаются широтой/долготой).
- Координаты устройства берутся из браузера (`navigator.geolocation`).
- Стаканчики и студенты создаются с небольшим смещением по широте/долготе от твоей позиции, поэтому они «висят» в пространстве рядом с тобой.

### Ограничения

- **HTTPS или localhost** обязательны (геолокация и камера в небезопасном контексте работают не везде).
- На **телефоне** точность и плавность лучше (GPS + компас). В помещении GPS может быть неточным.
- Рекомендуется **Chrome на Android**; в Firefox на мобильных есть ограничения Device Orientation API.

## Что использовано в проекте

| Элемент | Где |
|--------|-----|
| **a-entity** | Все объекты сцены (стаканчики, однокурсники, стены, пол, свет) |
| **a-sky** | Фон сцен ы с текстурой (360° изображение) |
| **Текстуры** | Дерево (пол), кирпич (стены) — CDN A-Frame |
| **3D-модель** | Стаканчик: примитивы в коде; опционально — Blender → `coffee_cup.glb` (см. BLENDER_GUIDE.md) |
| **Изображения** | Текстуры в `<a-assets>`, sky |
| **Аудио** | Фон (кофемашина), звук сбора (дзынь) — CDN A-Frame |
| **Текст** | UI: счёт, таймер, финальный экран, уровень Barista |
| **Земля/пол** | `<a-entity geometry="primitive: plane">` с текстурой дерева |
| **Свет** | Ambient + point + directional (тёплые оттенки) |
| **Камера** | `<a-camera>` с курсором и wasd-controls |
| **Анимация** | Вращение и «покачивание» стаканчиков; движение однокурсников (position, alternate) |
| **Интерактивность** | Курсор, raycaster, клик по объектам с `data-clickable` и `game-item` |
| **Анимация по событию** | Анимации заданы в разметке (loop, alternate) |
| **Физика** | aframe-physics-system: dynamic-body у стаканчиков, static-body у пола |
| **Трансформации предок/потомок** | Контейнеры `#walls`, `#game-objects` — потомки наследуют положение/масштаб |
| **AR (маркер)** | AR.js: маркер (Hiro или pattern), сцена в `ar.html` |
| **AR (GPS)** | AR.js location-based: камера и объекты по широте/долготе, сцена в `ar-gps.html` |

## Краткий чеклист запуска

1. Поднять локальный сервер в папке `coffee-game`.
2. Открыть **http://localhost:8080** → игра.
3. Открыть **http://localhost:8080/ar.html** → AR по маркеру; **http://localhost:8080/ar-gps.html** → AR по GPS.
4. Для маркера: распечатать Hiro (или создать pattern + .patt), разрешить камеру и навести на маркер — появится мини-кофейня.
5. Для GPS: разрешить камеру и геолокацию — объекты появятся вокруг тебя по координатам.
6. При желании сделать свою модель стаканчика — следовать **BLENDER_GUIDE.md** и скрипту `blender_coffee_cup.py`.
