# Построчное объяснение index.html и ar-gps.html

## index.html — десктопная игра «Кофейня»

### Строки 1–6: начало HTML
- **1** `<!DOCTYPE html>` — объявление типа документа (HTML5).
- **2** `<html lang="ru">` — корневой тег, язык страницы русский (для доступности и поиска).
- **3** `<head>` — начало блока заголовка (метаданные, стили, скрипты).
- **4** `<meta charset="UTF-8">` — кодировка UTF-8, чтобы кириллица отображалась верно.
- **5** `<meta name="viewport" ...>` — настройка области просмотра на мобильных (масштаб, ширина).
- **6** `<title>...</title>` — заголовок вкладки в браузере.

### Строки 7–8: подключение библиотек
- **7** Подключение **A-Frame** — фреймворк для 3D/VR в браузере (сцена, камера, объекты).
- **8** Подключение **aframe-physics-system** — физика (static-body, dynamic-body), в игре используется для пола (static-body).

### Строки 9–82: стили CSS
- **9** `<style>` — начало блока стилей.
- **10** Стили для `body`: убираем отступы, скрываем прокрутку, шрифт.
- **11–16** `#ui-overlay` — панель сверху (очки, таймер). `position: fixed` — поверх сцены; `pointer-events: none` — клики проходят к сцене; по умолчанию `display: none`, показывается при старте игры.
- **17–23** `#top-bar` — внутренний блок панели: flex, фон, скругления, тень.
- **24** Цвет чисел очков и таймера (золотой).
- **25** Стиль блока комбо (размер, цвет, выравнивание).
- **27–30** `.float-pts` — всплывающие «+5» / «-5»: фиксированная позиция, анимация.
- **31** Класс `.good` — золотой цвет для плюсовых очков.
- **32** Класс `.bad` — красный для минусовых.
- **34–37** `@keyframes floatup` — анимация: текст поднимается вверх и исчезает за 1 сек.
- **39–43** `#start-screen` — стартовый экран (меню): на весь экран, тёмный фон, flex по центру, z-index 150 чтобы быть над сценой.
- **44–46** Стили заголовка и текста на стартовом экране.
- **47** `#start-screen.hidden` — при добавлении класса `hidden` меню скрывается (`display: none`).
- **48** `.hint-grid` — сетка подсказок (две колонки).
- **49–53** `.hint-card` — карточка подсказки (рамка, фон, скругления).
- **54** Стиль иконки в карточке (если есть).
- **56–60** `#final-screen` — экран конца игры: по умолчанию скрыт (`display: none`), тёмный фон, z-index 200.
- **61** `#final-screen.visible` — при классе `visible` экран показывается (flex).
- **62–65** Стили заголовка, большого счёта и уровня на финальном экране.
- **66–69** `.stats-row` и `.stat-box` — ряд блоков статистики (поймано, промахов, комбо).
- **70–72** Стили числа и подписи в блоках статистики.
- **74–77** `.btn` — кнопки: отступы, фон, скругления, курсор.
- **78** Эффект при наведении на кнопку (цвет, лёгкий подъём).
- **79–80** Класс `.btn-ar` — кнопки перехода в AR (другой цвет, отступ).
- **81** Закрытие `</style>`.

### Строки 83–98: сцена A-Frame
- **84** `<body>` — тело страницы.
- **86** `<a-scene>` — корень 3D-сцены A-Frame.
- **87** `cursor="rayOrigin: mouse; fuse: false"` — курсор следует за мышью, без режима «fuse» (удержание).
- **88** `raycaster="objects: [data-clickable]; far: 25"` — луч проверяет пересечения только с объектами, у которых есть атрибут `data-clickable`, на расстоянии до 25 единиц.
- **89** `renderer="antialias: true; colorManagement: true"` — сглаживание и управление цветом.
- **90** `shadow="type: pcfsoft"` — мягкие тени.
- **91** `vr-mode-ui="enabled: false"` — кнопка входа в VR скрыта.
- **92** `physics="debug: false"` — физика включена, отладочный режим выключен.

### Строки 94–99: ресурсы (a-assets)
- **94** `<a-assets timeout="20000">` — блок предзагрузки ресурсов, таймаут 20 сек.
- **95–96** Текстуры пола и стен (дерево, кирпич) — URL с CDN A-Frame.
- **97** Текстура неба (панорама 360°).
- **98** 3D-модель стаканчика (GLB из Blender).
- **99** Закрытие `</a-assets>`.

### Строки 100–114: небо, свет, камера

- **101** Ambient light — рассеянный свет по всей сцене (тёплый оттенок).
- **102–103** Два point light — точечные источники для объёма и теней.
- **104** Directional light — направленный свет (как солнце).
- **106** `<a-entity id="rig">` — контейнер камеры, позиция «глаза» игрока (1.6 по Y, 6 по Z).

- **108** `<a-cursor>` — визуальный курсор в 3D, реагирует только на `[data-clickable]`, цвет и материал.
- **109–110** Закрытие rig и камеры.

## ar-gps.html — AR-игра по GPS

### Строки 188–212: координаты и фолбэк
- **188–189** Демо-координаты (Москва), если GPS недоступен.
- **191–195** `setCoords(lat, lon, demo)` — сохраняем userLat/userLon, выводим в #coord-val, при demo добавляем «(демо)»; если игра ещё не стартовала — запускаем startGame().
- **197** `useFallback()` — вызывается при ошибке GPS или таймауте.
- **198** Не запускаем фолбэк, если игра уже идёт.
- **199** Сообщение пользователю.
- **200–202** Если есть камера и компонент gps-camera — задаём simulateLatitude/simulateLongitude демо-координатами, чтобы камера и объекты были в одной «точке мира».
- **203** Вызов setCoords с демо-координатами.
- **205–207** Слушатель события gps-camera-update-position (AR.js): при обновлении позиции камеры по GPS вызываем setCoords с реальными координатами.
- **209** Проверка поддержки геолокации.
- **210–213** watchPosition: при успехе — setCoords с реальными координатами, при ошибке — useFallback(); опции: высокая точность, таймаут 10 сек.
- **214** Если геолокации нет — сразу фолбэк.
- **216** Через 12 сек, если игра так и не стартовала (нет GPS и не сработал watchPosition) — принудительный фолбэк.

### Строки 218–288: обработка тапов (клик по объектам в AR)
- **218** После события loaded сцены (когда A-Frame и AR.js готовы).
- **219–221** canvas сцены, объект Three.js сцены (object3D), рейкастер и вектор для нормализованных координат (NDC).
- **223–226** `getPointerCoords(evt)` — из события мыши или тача достаём clientX/clientY.
- **228–232** `toNDC(screenX, screenY)` — переводим экранные координаты в NDC (-1…1) для Three.js Raycaster.
- **234–243** `findClickableEntity(threeObject)` — от 3D-объекта поднимаемся по родителям и ищем A-Frame entity с классом clickable.
- **245** `onPointerTap(evt)` — обработчик тапа/клика.
- **246–247** Получаем координаты и переводим в NDC.
- **249–251** Находим камеру и её Three.js камеру (нужна для луча).
- **253** Строим луч из камеры в направлении NDC.
- **254** Пересечения луча со всей сценой (true = с детьми).
- **256–261** По пересечениям ищем первый clickable entity и вызываем у него событие click (чтобы сработали обработчики на стаканчике/студенте).
- **263–264** Вешаем onPointerTap на click и touchstart по canvas (touch с preventDefault для корректной работы на мобильных).
- **289–291** Закрытие script, body, html.

Итог: в **index.html** — обычная 3D-игра в браузере (камера, пол, стены, стаканчики по позициям x,y,z). В **ar-gps.html** — та же механика, но камера и объекты привязаны к GPS: камера — к устройству, объекты — к координатам lat/lon через gps-entity-place, поэтому стаканчики и студенты появляются «в мире» вокруг тебя, а не под камерой.
